個人開発演習の概要（開発要件）

目的: 研修3ヶ月目までに学んだ講義内容の理解度、及び実装力を測ること。

内容: 社員の情報を一元管理（検索、登録、変更、削除）する Web アプリケーションの開発

演習期間: 8日間(アプリ開発7日, 成果報告会報告会準備1日)


演習形式:

予め提示されたシステム要件に沿って実装を行う。

要件機能の実装を終えたら、独自機能の実装を行う。

機能・使用技術

要件機能

機能名 内容 補足
認証機能 ログイン・ログアウトが可能 未ログイン時はログイン画面のみアクセス可能
権限管理機能 管理者・一般の二段階 権限差で操作可能な機能を制限
表示機能 社員一覧を表示 権限による表示情報制限
検索機能 名前検索・部署名検索が可能 権限による表示情報制限
CRUD 登録・更新・削除が可能 一般ユーザーは自身の更新のみに制限

独自機能

機能名 目的 実装内容 使用技術
権限に応じたデザイン変更 誤操作・誤共有の防止 / UX向上 管理者（権限2）でヘッダー/フッター/サイドバー/文字色を切替 HTML(Thymeleaf) / CSS
パスワード表示切替・確認フォーム 誤登録・誤送信の防止 表示切替（password⇄text）＋確認入力の一致チェック＋Enter送信無効化 JavaScript
パスワードのハッシュ化 漏洩時リスク低減 パスワード暗号化処理の実装、暗号化パスワードに対応したログイン照合機能の実装。DB制約を16charから255charに変更 Java / Spring Security(Bcrypt)
非同期ページング 視認性向上 / HTML肥大化防止 検索結果をページ分割し、Ajaxで一覧のみ部分更新 JavaScript(Ajax)/ Spring(Pageable)
CSV出力/入力 Excel連携 / 一括更新 / 保守性向上 Excel対応CSVの出力、CSV読込で一括登録・更新＋一括バリデーション Java / 外部ライブラリ(OpenCSV)

使用技術一覧

バックエンド

  • Java 17
  • Spring Boot 3.1.4
  • Spring MVC（Webアプリ/API）
  • Spring Data JPA（DB操作）
  • Bean Validation（入力バリデーション）
  • Spring Security（BCryptハッシュ化）
  • OpenCSV（CSV入出力・パース）

View / フロント

  • HTML(Thymeleaf) / CSS
  • JavaScript / Ajax（部分更新）

データベース

  • Oracle Database（本番想定）
  • H2 Database（開発/テスト）

開発・テスト

  • Maven（依存管理）
  • Spring Boot DevTools
